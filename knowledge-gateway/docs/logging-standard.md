# Knowledge Gateway Logging Standard

## Purpose
Define one canonical way for coding agents to interpret logging intents and write to the gateway.

## Intent Mapping
- "log this session" -> `log_coding_session`
- "log meeting" -> `log_meeting`
- "log decision" -> `log_decision`
- "read a note" -> `get_obsidian_note`
- "write custom note" -> `upsert_obsidian_note` (manual/exception-only)

## Session Logging Contract
Use `log_coding_session` for routine work updates.

### Required Fields
- `employer_name`
- `project_name`
- `session_title`
- `idempotency_key`
- `started_at` (ISO-8601)

### Optional Fields
- `ended_at` (ISO-8601, must be >= started_at)
- `objective`
- `summary`
- `thought_process`
- `methodology`
- `major_changes`
- `advantages`
- `disadvantages`
- `blockers`
- `next_steps`
- `learnings`
- `skills_updates`
- `tags` (list)
- `source` (for example: `codex`, `claude-code`, `cursor`, `gemini-cli`)

### Idempotency Key Rules
- Regex: `^[A-Za-z0-9._:-]{6,128}$`
- Recommended shape: `<agent>-<machine>-<utcstamp>-<seq>`
- Retry behavior: reuse same key for the same logical session write.

## Canonical Note Structure
Generated by gateway for `log_coding_session`:
- heading + metadata block
- sections:
  - Objective
  - Summary
  - Thought Process
  - Methodology
  - Major Changes
  - Advantages
  - Disadvantages
  - Blockers
  - Next Steps
  - Learnings
  - Skills Updates

## Path Convention
- `Employers/<Employer>/<Project>/<YYYY-MM-DD-update-<feature-slug>>/session-<session-id>.md`
- Similar convention used for meeting/decision notes.

## Policy
- Prefer structured logging tools over free-form note writes.
- Avoid `upsert_obsidian_note` for routine session/meeting/decision logs.
- Destructive operations are disabled; archive is the deletion model.

## Examples

### Session
```json
{
  "name": "log_coding_session",
  "arguments": {
    "employer_name": "Benori",
    "project_name": "Project Alpha",
    "session_title": "Fix MCP Host Header Validation",
    "idempotency_key": "codex-laptopA-20260411T201000Z-01",
    "started_at": "2026-04-11T20:10:00Z",
    "summary": "Resolved Cloudflare Access + 421 host-header mismatch.",
    "next_steps": "Document onboarding for all coding agents.",
    "source": "codex"
  }
}
```

### Meeting
```json
{
  "name": "log_meeting",
  "arguments": {
    "employer_name": "Benori",
    "project_name": "Project Alpha",
    "meeting_title": "Gateway Security Review",
    "meeting_datetime": "2026-04-11T20:45:00Z",
    "summary": "Confirmed Access + app-level gate strategy."
  }
}
```

### Decision
```json
{
  "name": "log_decision",
  "arguments": {
    "employer_name": "Benori",
    "project_name": "Project Alpha",
    "title": "Use dual-layer auth for MCP",
    "rationale": "Cloudflare Access + app API key gives defense in depth.",
    "status": "active"
  }
}
```
